Ниже представлены подробные ответы на третий блок вопросов: **«Теоретические основы реляционной модели»**. Этот блок является фундаментом для понимания того, как работают современные базы данных (SQL).

---

### 1. Общие понятия реляционного подхода, основные концепции и термины

Реляционный подход (от англ. *relation* — отношение) основывается на представлении данных в виде совокупности таблиц.

**Основные термины:**
*   **Отношение (Relation):** Это двумерная таблица.
*   **Кортеж (Tuple):** Строка таблицы. Представляет собой запись о конкретном объекте.
*   **Атрибут (Attribute):** Столбец таблицы. Определяет свойство объекта.
*   **Домен (Domain):** Набор допустимых значений для одного или нескольких атрибутов (например, домен «Возраст» — это целые числа от 0 до 120).
*   **Схема отношения:** Список имен атрибутов с указанием их доменов.

**Ключевые концепции:**
*   **Первичный ключ (Primary Key, PK):** Один или несколько атрибутов, которые **однозначно** идентифицируют каждый кортеж (строку) в таблице. Ключ должен быть уникальным, и его значения не могут быть пустыми (NULL).
*   **Внешний ключ (Foreign Key, FK):** Это атрибут одной таблицы, который содержит значения первичного ключа другой таблицы. С помощью FK реализуются связи между таблицами и обеспечивается *ссылочная целостность*.
*   **Индексы:** Специальные структуры во внешней памяти, предназначенные для ускорения поиска данных. Работают по принципу алфавитного указателя в книге: СУБД сначала ищет значение в индексе, а затем переходит по ссылке к нужной строке в таблице.

### 2. Изложите основные свойства отношений

Согласно теории Эдгара Кодда, отношение (таблица) должно обладать следующими свойствами:

1.  **Отсутствие дубликатов кортежей:** В таблице не может быть двух абсолютно одинаковых строк (это обеспечивается наличием первичного ключа).
2.  **Атомарность значений:** В каждой ячейке таблицы должно находиться только одно элементарное значение (никаких списков или массивов внутри ячейки). Это требование первой нормальной формы.
3.  **Уникальность имен атрибутов:** В рамках одной таблицы столбцы не могут называться одинаково.
4.  **Порядок кортежей не важен:** Логика системы не должна зависеть от того, в какой последовательности записаны строки.
5.  **Порядок атрибутов не важен:** Столбцы можно менять местами без потери смысла данных.
6.  **Однородность данных в столбце:** Все значения в одном столбце должны принадлежать к одному домену (типу данных).

### 3. Базисные средства манипулирования реляционными данными. Теоретико-множественные операции реляционной алгебры

Реляционная алгебра — это теоретический процедурный язык. Операции принимают на вход отношения и выдают на выходе новое отношение.

**Теоретико-множественные операции** (требуют совместимости схем таблиц — одинаковое количество и типы столбцов):

1.  **Объединение (Union, $\cup$):** Результат содержит все кортежи, которые есть либо в первой, либо во второй таблице (без повторов).
2.  **Пересечение (Intersection, $\cap$):** Результат содержит только те кортежи, которые присутствуют одновременно в обеих таблицах.
3.  **Разность (Difference, $-$):** Результат содержит кортежи, которые есть в первой таблице, но отсутствуют во второй.
4.  **Декартово произведение (Product, $\times$):** Комбинирует каждую строку первой таблицы с каждой строкой второй таблицы. Если в таблице А — $m$ строк, а в Б — $n$ строк, в результате будет $m \times n$ строк.

---

### 4. Специальные реляционные операции реляционной алгебры

Эти операции разработаны специально для работы со структурами таблиц:

1.  **Выборка (Selection, $\sigma$):** Горизонтальное подмножество. Выбирает строки, удовлетворяющие заданному условию (аналог `WHERE` в SQL).
2.  **Проекция (Projection, $\pi$):** Вертикальное подмножество. Выбирает только определенные столбцы из таблицы и удаляет дубликаты, если они возникли (аналог `SELECT DISTINCT column_name`).
3.  **Соединение (Join, $\bowtie$):** Самая важная операция. Сопоставляет строки двух таблиц на основе общего атрибута (например, по ID). Существуют разные виды: эквисоединение, естественное соединение, внешние соединения.
4.  **Деление (Division, $\div$):** Используется для запросов типа «найти объекты, которые связаны со **всеми** объектами из другой таблицы» (например, «найти студентов, которые сдали все экзамены»).

### 5. Охарактеризуйте особенности реляционного исчисления

Реляционное исчисление — это альтернативный подход к манипулированию данными. В отличие от алгебры (которая говорит «КАК» получить результат, т.е. процедурный подход), исчисление является **декларативным** («ЧТО» мы хотим получить).

**Ключевые особенности:**
*   Основано на логике предикатов первого порядка.
*   Запрос формулируется как логическая формула, описывающая свойства искомых данных.
*   **Виды исчисления:**
    1.  *Исчисление кортежей:* Переменные принимают значения строк таблицы. Именно этот подход лег в основу языка SQL.
    2.  *Исчисление доменов:* Переменные принимают значения из доменов (столбцов). Легло в основу языка QBE (Query By Example).
*   **Кванторы:** В исчислении активно используются квантор существования ($\exists$ — «существует хотя бы один») и квантор всеобщности ($\forall$ — «для всех»).
*   **Эквивалентность:** Доказано, что реляционная алгебра и реляционное исчисление эквивалентны по своей выразительной мощности (любой запрос на одном языке можно переписать на другой). Это свойство называется «реляционной полнотой».
